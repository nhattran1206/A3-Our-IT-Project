Project description



I. Overview


1. Topic

For this project, we propose to create a platform that users can freely access to discuss about their favorite movie and get recommendation for new movies to watch. Currently, there have been too many movies produced, and the production rate doesn’t seem to drop. That’s why, there is a high demand for a platform that can recommend the user which movie that can fit the user’s movie taste.

For the outcome of the project, we hope that after the next 12 weeks, we will be able to create a complete website interface with every intended feature implemented into it. The website will be hosted through a credible third company so that the user can access easily and the website loading speed won’t be affected the high rate of user’s access. Furthermore, the website system must be secured from outsider threat, the user’s private data must be encrypted and store carefully in the server’s database, so that it won’t be stolen by Internet pirate.



2. Motivation

The motivation that helps us to create this website is due to our own boredom while searching for new movie to watch. Usually, it will take us hours of our precious leisure time to search for a movie to watch. Therefore, we planned to create this website not only to help other who had faced this type of problem like us, but it is also to help us find a good film to watch ourselves. In our opinion, this project will be a very fun experiment because the website’s movie rating won’t be affected by us, but it is affected by the community of user who contributed in judging if a specific movie is worth to watch or not. That’s why, we would love to see our project to come to fruit and experience the result ourselves.



3. Landscape

It is unarguable that our current project isn’t the first movie rating website to exist, there are many other movie rating websites that has been existing way before our project. Sites such as IMDB or Rotten Tomato are 2 of the names that will appear in many people mind when asked about popular movie rating website. Despite all odds again us, we still think that our movie rating website will leave special impression in our user since our website care more about the user and also influenced by the community more.

To accomplish the aim that we have set out at the start of the project, we have some goal that we need to finish. First of all, we need to focus in designing and creating a approachable and professional website layout first. The website design needed to be professional and simple so that the user can understand that we value their experience and wanted to create a place for them to search for serious and credible reviews. To be able to achieve that, color scheme act as the most important factor. Color psychology is the science of how color patterns can affect the user’s choices and behavior[1]. In other word, a single-color change can affect the likeliness of a user continuing to access the page again.


Secondly, another function of the website that we need to take account of is the login sequence of the website. Our goal is to create a smooth and simple login and sign-up sequence that the user can handle comfortably. The user should be able to input their username and password easily and submit it to the server database without fearing that their data being leak or stolen. If the user hasn’t created any account yet, they should be able to click on a button to switch to a sign-up page and create an account using a username and a password easily.


Thirdly, an overachieving goal that we planned to do is to install into our website system a member ship program system. This system act like a meter of credibility of a user, this meter will determine if the user’s reviews of the movies is trustworthy or not. This system is very important to create a platform that the other user can know which review to believe in when they are browsing throw each movie.


Finally, the last goal that needed to be done is to create a personal list of movies to recommend for the user. As mentioned above, another function that we aim to achieve for our website is its ability to automatically create and recommend the user a list of movies that may fit into their liking. Therefore, we will need to implement an “Suggest” algorithm that can collect the user’s favorite genre and recommend the movies that both have good rating from other user, and with the same genre.


2. Plans and Progress

This project would have basic features of a film reviewing website, including sign up and sign in system, membership system, film and review filter, search engine, professional critics section and ticket scanner system. We began by creating our basic website design with HTML and CSS. We then hosted an online website using GitHub. The home page of the website has been complete and functions well. However, other features of the website are under prototype state. The database is planned to be built using database software such as MySQL.


In order to maintain the reputation of the website as well as avoiding spammed reviews and comments, we planned to limit the account registered by using phone number verification. Every phone number can only be used for one account only, and if users need to create a new one, the old account must be deleted. A problem arises that if the user processes multiple phone numbers, they can sign up as many new accounts as they desire. We are still finding a solution to the issue. Registering using citizen ID cards can seem to solve trouble; however, this method would require cooperating with the government, which would be impossible for our university project. Users without accounts could only read the reviews and comments rather than contributing to the website. In order to build a sign-up system, we would first have to create a database. Then, by connecting frontend and backend together using backend technology, all information received could be sent to the database. Moreover, coding for validation would be applied to check if the input data is correct or not. Finally, after users enter their login information, those would be verified with the ones in the database. Making phone number verification is also a problem that we must solve to complete the project. Online tutorial videos provide detailed steps on making phone number verification via Firebase, which we plan to make after finishing the website designing. Phone number verification is an important aspect of collecting customers’ data, which is crucial for the development of many projects.


Having an account is crucial to process with the membership system. This is a system to decide the reliability of an account through user’s voting. Every comment and review would have an up vote and a down vote button, where users can decide whether that review could be trusted or not. The difference between the numbers of up votes and down votes of a review as well as the number of reviews that are placed in the top review list of a film would be converted into credibility points. Credibility point is a system that would affect membership rank. The higher the membership rank is, the more privilege the user can experience. These benefits would include but not limited to a trial of an ad-free version, having reviews pinned on top of the page and displaying rank-exclusive badges. Users can check their progress of accumulative credibility points in the Personal Information interface as well as adjusting any information if needed. The membership system could use the same database as the sign-up system. All credibility points would be calculated as soon as possible. We would have to test whether our technology could do the calculation every hour, and then try to develop it so that credibility points could be updated right after adjustment happens.


A film filter plays a significant role in providing an appropriate film list for users. Each movie would be assigned tags with their corresponding genres as well as the age limitation. Users could choose the tags that they prefer while also excluding the genres that they dislike.


Aside from film filtering, a search system is used to find film by name. We wish to maximize the potential of the search engine. For title search, we would displace the search results based on the numbers of keywords that match the search in a correct order. After that, results with matched keywords but with different order would be shown. We planned to use engines such as Elastic Search to develop our search engine. The database would be updated frequently to ensure that new films and movies are added to our website on time. We would have to recruit a data specialist to set up the whole database. That person would also oversee reviewing user search history to ensure if our search engine is suitable for certain queries.


Many popular film review websites have their own film critic team. These critics could attract more users since people are most likely to believe in experts in the field. Critics’ reviews would be pinned on top of the comment section with special borders for highlighting purposes. Unfortunately, it is hard to find critics that are willing to help with our project. Our plan is to run the website to gain attention until we can contact and collaborate with some film critics. Using advanced CSS techniques for highlighting purposes, we would be able to distinguish the critic section from normal users.


In this project, we want to introduce the ticket scanner. Having people who write reviews without watching the movie could lead to misleading understanding about the film. Our objective is to build a built-in scanner that would scan cinema tickets. Accounts with proof of ticket scanning would have their reviews highlighted with watched approval, which would be awarded with more credibility points. After a movie is broadcasted online, the ticket scanner for that movie would be closed in order to stop the reusing of tickets. One main problem that we encounter is how to create the scan system as well as accessing the cinema database for customers’ information. Most cinema tickets nowadays would have their corresponding barcodes or QR codes. Using Android Studio, we could create a code scanner app, and then if available, we would access the database of the cinema to ensure the reliability of the ticket. For tickets without any code, a text scanner could be used to identify the information written. However, this method would require full access to the database of cinemas to retrieve data about the ticket or else the text scanner would serve nothing. Our ambition is to combine both text and code scanner into one scanner that could directly read the database to validate the ticket. It is not an easy target since they use difference engine and coding.


We pay attention to keeping a healthy environment for film lovers. Swear words are prohibited in every section of the website. Reviews and comments would go through a “profanity check” before being published. Swear words would automatically be omitted from the review using the If-in logic. However, if an account is found to have a negative attitude, which is swear words for multiple reviews and comments, it would receive a strike. The accumulation of three strikes would result in a complete ban. With the use of Python, Flask and GraphQL, we would manage to create a profanity filter that could scan through the comments and reviews for any swear words. We also need to create a separate database to store the profanity list as well as optimizing the filter in case people find cheating methods.


In order to keep our users away from being spoiled, we have decided to build a separate section for spoilers. Reviews and comments on non-spoiler page would go through the “spoiler check” just like with the “profanity check.” Reviews that violate the rule would receive the same punishment as swear words. A spoiler database is created to store the keywords of each movie. The “spoiler check” would use the same system with the “profanity check”, but had the database replace with the spoiler database.

6. Testing

Since our group is not a professional at web programming, we are only thinking about website functionality testing and security testing. First, we do a links test since we need to confirm that any link that leads to our website works properly and there are no broken links or any link that leads to the wrong website or wrong page. Secondly, we test the performance of the website. We need to confirm that our login page works efficiently, there are no errors that occur when our customers want to create a login account or try to login in their account and we need to confirm that our customer can see other people review on our website. Thirdly, we will do the HTML and CSS test where we test if there are any HTML syntax errors and we need to verify that our site is available for the search machines. Finally, we will do the security testing, in this test we will simulate a malicious source attack to evaluate our website security and we need to making sure that there is no login ability if people enter wrong username or password and lastly, we also need to ensure that there are no restrict files are downloadable without appropriate access.

8. Risks

Our project is about developing and publishing a website, and it is not always easy. One of the most usual challenges that could be an internal risk for our group and project is difficulties in learning to design and build the website in terms of UX UI Design, programming language (HTML, CSS, JS), database. Time is important in such a project as ours, and lack of experience in designing and building websites is certainly a threat to the general progress of the group. There will be copyright problems when our website is about reviewing and evaluating films. There will be thousands of other competitors that want to demolish our website, and so nothing is simpler than accusing us of copying ideas from their websites. Another risk is that this website will be vulnerable before cyber-attacks because we do not have a strong security for our data yet. Finally, it is the risk in our web services, which is the problem with the scanner system for checking the credibility of reviewers. Particularly, we will have to expect delays and crashes in response between the server and the clients because our website’s data storage is inadequate – we do not have enough funds to upgrade the database.
